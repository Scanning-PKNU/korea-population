# Contributing to korea-population

저희 프로젝트에 관심 가져주셔서 감사합니다!  
이 문서는 이 저장소에 기여하고 싶을 때 따라주면 좋은 절차와 규칙을 안내합니다.

---

## 🎯 기여 방식  
- 버그 수정 (데이터 오류, 전처리 버그, 웹 UI 오작동 등)  
- 새로운 데이터 추가 (예: 새로운 도시/지역 population 데이터)  
- 문서 개선 (README, 설명 문서, 주석 등)  
- 기능 추가 또는 개선 (예: 데이터 필터 / 통계 / 시각화 기능)  

---

## 🐞 Issue 열기  
- 새로운 기능 제안 또는 버그는 먼저 Issue를 열어주세요.  
- 이미 유사한 이슈가 없는지 검색해 주세요.  
- 가능하면 문제 상황을 명확히 설명하고, 재현 방법 또는 데이터 예시를 포함해 주세요.  

---

## 🌿 브랜치 & 커밋 규칙  
- 브랜치명 예시  
  - 기능 추가: `feature/이름`  
  - 버그 수정: `fix/설명`  
  - 문서 작업: `docs/무엇을_했는지`  

- 커밋 메시지는 가능한 한 명확하게 써 주세요.  
  - 형식 예시 (의무는 아님):  
    - `feat: add new city population data`  
    - `fix: correct Haman population number`  
    - `docs: update README with dataset description`  

---

## 🔀 Pull Request 제출  
- PR 전에 반드시 fork → branch → 변경 → 커밋 → PR 순서 따라 주세요.  
- PR 설명에는 변경 내용, 이유, 영향 등을 명확히 적어 주세요.  
- 가능하면 데이터 추가나 변경 시 before/after 비교 (예: 통계값 변화, CSV 예시) 포함해 주세요.  
- 기존 코드 구조/스타일을 유지해 주시면 리뷰와 유지보수가 용이합니다.  

---

## 🧰 코드 스타일 & 파일 구조  
- 데이터 파일은 `.csv` 형식으로 유지하고, 파일명은 읽기 쉽게 작성해주세요 (예: `cityname_data.csv`).  
- 스크립트 파일은 Python을 사용하며, 함수/변수명은 영어권 코딩 관습을 따르는 것이 좋습니다.  
- HTML / 웹 UI 부분은 기존 구조를 크게 변경하지 않는 것을 권장합니다.  
- 문서 수정 시 Markdown 문법 사용, README.md 혹은 추가 문서 파일로 작성  

---

## 🙏 기여 감사 & 라이선스  
- 모든 기여는 환영합니다. 다만, 커밋 또는 PR 제출 시 본인이 해당 변경에 대해 책임지고, 기존 데이터나 기능을 임의로 변경할 경우 주의 부탁드립니다.  
- 저장소 루트에 LICENSE 파일이 있다면(또는 추가 예정이라면), 해당 라이선스 조건에 따라 코드를 사용/배포/기여해 주세요.
